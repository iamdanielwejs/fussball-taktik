# Definition of Ready (DoR)

Ein Product Backlog Item ist **Ready** fuer einen Sprint, wenn alle folgenden Kriterien erfuellt sind:

## Kriterien

1. **User Story formuliert** - Das PBI ist als User Story geschrieben ("Als ... moechte ich ... damit ...")
2. **Akzeptanzkriterien definiert** - Mindestens 3 konkrete, testbare Akzeptanzkriterien sind vorhanden
3. **Story Points geschaetzt** - Das Team hat die Komplexitaet in Story Points bewertet
4. **Abhaengigkeiten geklaert** - Alle technischen und fachlichen Abhaengigkeiten sind identifiziert und aufgeloest
5. **Design-Skizze vorhanden** - Fuer UI-relevante Stories liegt eine grobe Skizze oder Beschreibung vor
6. **Klein genug** - Das PBI kann innerhalb eines Sprints abgeschlossen werden (max. 13 SP)
