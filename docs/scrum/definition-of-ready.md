# Definition of Ready (DoR)

A Product Backlog Item is **Ready** for a sprint when all of the following criteria are met:

## Criteria

1. **User story written** - The PBI is written as a user story ("As a ... I want ... so that ...")
2. **Acceptance criteria defined** - At least 3 specific, testable acceptance criteria are present
3. **Story points estimated** - The team has estimated the complexity in story points
4. **Dependencies clarified** - All technical and domain dependencies are identified and resolved
5. **Design sketch available** - For UI-related stories, a rough sketch or description is provided
6. **Small enough** - The PBI can be completed within a single sprint (max 13 SP)
